面向对象的基础知识点

面向对象的三大特性

封装、继承和多态

对象、引用和指针的区别

Java语言中的引用变量存储在栈中，相当于C语言中的指针，存放引用的数组、对象的地址值，而被引用的实际值存储在堆中。

this关键字

this关键字最大的作用就是让类中一个方法可以访问该类里另一个方法或实例变量。

补充内容

  1. Java类名采用驼峰原则：即每个单词首字母大写，其他字母小写，中间无分隔符，如：compareAndSwap(CAS操作);

  2. static修饰的成员不能访问没有static修饰的成员；(底层与内加载过程有关，简单来说static修的成员在类中称为静态变量，是在类加载阶段就分配好内存并初始化，此时没有static修饰的成员变量还没有进行该操作，所以不能访问)；

  3. 通过new关键字来调用构造器，从而创建实例(一种饿汉式创建类的方法，即其他类显示调用构造器进行类的创建，此方法也是也是大多数创建类实例的方法，相对应的是懒汉式创建类的方法，即把该类的构造器用关键字private封装起来，不允许外部类调用，同时提供一个public方法供外部类调用来创建实例，最典型的应用是单例模式)，该过程的底层原理主要分为5个步骤

    1. 判断类加载过程，如果执行对象创建之前没有进行过类加载，首先执行类加载过程(加载、连接(验证、准备、解析)、初始化、使用、卸载)。

    2. 为成员变量分配内存空间，采用何种方法来分配内存空间取决于Java堆中内存的完整性，如果采用的是带有空间压缩性质的垃圾回收器(如 Serial、SerialOld、G1)，则利用指针碰撞的方法来进行变量内存分配，如果使用的垃圾回收期没有该性质(如CMS)，则利用空闲列表的方法进行内存分配。

      指针碰撞：通过指针的偏移量来代表分配给变量的内存大小，指针的一边是使用过的内存、另一边是没有使用的内存；

      空闲列表：通过记录有Java堆上空闲内存的列表来给变量分配内存。

    3. 变量的初始化操作，给相应类型的变量赋予初值；

    4. 设置对象头，对象头的内容包括hashcode值、锁标志位等；

    5. 调用Class文件中的()方法。

类方法分析

方法的所属性

方法不能独立于类或对象而存在，如果这个方法使用了static修饰，则这个方法属于这个类，否则这个方法属于这个类的实例。

  1. 方法不能独立定义，方法只能在类体力定义；
  2. 从逻辑意义上来看，方法要么属于该类本身，要么属于该类的一个对象；
  3. 永远不能独立执行一个方法，执行方法必须使用类或对象作为调用者。

形参个数可变的方法

public static void test(int a, String... books);
public static void test(int a, String[] books);

上面两种方法都可以定义形参个数可变的方法，但是调用的时候略有差别java

test(5, "疯狂Java讲义", "并发编程")；
test(5, String[]{"疯狂Java讲义", "并发编程"})

很显然利用第一种方法定义形参可变的方法在调用的时候更便捷

方法重载

如果有两个方法的方法名相同，但参数不一致，可以说一个方法是另一个方法的重载，具体情况如下：

  1. 方法名相同；
  2. 方法的参数类型，参数个数不一样；
  3. 方法的返回类型可以不相同；
  4. 方法的修饰符可以不相同；
  5. main方法也可以被重载。

方法的返回值类型不能用于区分重载的方法。

方法重写

方法的重写要遵循“两同两小一大”规则，“两同”即方法名相同、形参列表相同；“两小”指的是子类方法返回值类型应比父类方法返回值类型更小或相等；“一大”指的是子类方法的访问权限应比父类方法的访问权限更大或相等。

覆盖方法和被覆盖方法要么都是类方法（即用static关键字修饰），要么都是实例方法，不能一个是类方法，一个是实例方法。

方法重载和方法重写的区别

重载主要发生在同一个类的多个同名方法之间，也可能发生在父类方法和子类方法之间；重写发生在子类和父类的同名方法之间。

补充内容

成员变量与局部变量

成员变量是在类里定义的变量，局部变量指的是在方法里定义的变量。

构造器重载

使用this调用另一个重载的构造器只能在构造器中使用，而且必须做欸构造器执行体的第一条语句。

构造器执行顺序

父类构造器总会在子类构造器之前执行， 创建任何对象总是从该类所在继承树最顶层类的构造器开始执行，然后依次向下执行，最后才执行本类的构造器。

创建字符串的两种方法及原理

两种方法

String a = "Hello";
String a = new Stirng("Hello");

原理分析

当Java程序直接使用形如”Hello”的字符串直接量（包括可以在编译时就计算出来的字符串值）时，JVM将会使用常量池来管理这些字符串；当使用new String(“Hello”)时，JVM会先查看常量池中是否有”Hello”的字符串常量，如果有则只需调用String类的构造器来创建一个新的String对象，新创建的String对象被保存在堆内存中，如果没有该字符串常量，则会先在常量池中创建字符串直接量再执行上述操作，即该命令可能会创建1个或2个对象。

创建不可变类需要注意的点

设计一个不可变类，需要注意其引用类型的成员变量，如果引用类型的成员变量是可变的，需要采取一定的措施来保证该成员白能量引用的对象不会被改变，具体参见如下代码：

class Name{
    private String firstName;
    private String lastName;
    public Name(){}
    public Name(String firstName, String lastName){
        this.firstName = firstName;
        this.lastName = lastName;
    }
    //省略firstName和lastName的setter和getter方法
}

class Person{
    private final Name name;
    public Person(Name name){
        this.name = name;
    }
    public Name getName(){
        return name;
    }
}

public class Test{
    public static void main(String[] args){
        Name n = new Name("悟空", "孙")；
        Person p = new Person(n);
        //以下会输出悟空
        System.out.println(p.getName().getfirstName());
        n.setFirstname("八戒");
        //以下会输出八戒，改变了不可变类的性质，应对Person类重写
        System.out.println(p.getName().getfirstName());
    }
}
class Person{
    private final Name name;
    public Person(Name name){
        this.name = new Name(name.getfirstname(), name.getlastName());
    }
    public Name getName(){
        return new Name(name.getfirstName(), name.getlastName());
    }
}
//改变以后再执行main方法，第二次仍然会输出悟空。

final方法不可以被子类重写，但可以被重载；final类不可以被继承，final变量的值不可以改变，但final引用指向的对象的值可以改变。
