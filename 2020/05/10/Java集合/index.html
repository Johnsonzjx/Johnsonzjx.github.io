Java 集合

如果访问List集合中的元素，可以通过元素的索引来访问；如果访问Map集合中的元素，可以根据每项元素的Key值来访问其value；如果访问Set集合中的元素，只能根据元素本身来访问。

遍历集合的几种方式

调用forEach方法

因为Collection的父接口时Iterable，可以调用foreach方法(引用Predicative类，该类是函数时接口)，利用Lambda表达式可以实现

public class Test{
    public static void main(String[] args){
        Collection<Stirng> books = new Collection<>();
        books.add("深入理解Java虚拟机");
        books.add("Java并发编程实战");
        books.add("算法与数据结构");
        books.forEach((obj) -> System.out.println("书名为："+obj));
    }
}

利用Iterator

利用Iterator，内部含有hasNext()、Next()方法进行遍历访问

Iterator it = books.iterator();//注意生成迭代器以后不允许再改变集合books，否则会引起报错java.util.ConcurrentModificationException
while(it.hasNext()){
    String book = (String)it.next();
    System.out.println(book);
    if(book.equals("")){
        it.remove(); //可以删除Collection的" "元素
        //错误写法
        books.remove(book);
    }
}

利用Iterator类的方法

利用Iterator类里的forEachRemaining(Consumer action)方法，结合Lambda表达式实现

Iterator it = books.iterator();
it.forEachRemaining(obj -> System.out.println(obj));

HashSet和LinkedHashSet

LinkedHashSet是HashSet的一个子类，同样根据Hashcode值来决定元素的存储位置，同时使用链表维护元素的次序，看上去元素是以插入的顺序保存的，因此LinkedHashSet性能略低于HashSet的性能，二者底层的数据结构不同，HashSet和HashMap的底层结构相同，在Jdk1.7之前为数组+链表的数据结构，在Jdk1.8改为数组+链表+红黑树的数据结构，其中HashSet是HashMap把所有的value值置为null得到的，而LinkedHashSet多加了一个链表来维护元素次序。有关HashMap的源码分析放在另一篇博客专门介绍。

TreeSet的相关性质

TreeSet的数据结构

TreeSet是SortedSet接口的实现类，采用红黑树的数据结构来存储集合元素，支持自然排序和定制排序，调用compareTo(Object obj)方法来实现，该方法是Comparable接口下的方法，常用的实现该接口的类有：BigDemical、BigInteger和常用的数值型包装类；Character、Boolean、String、Data、Time。

compareTo方法

TreeSet是通过compareTo方法来判断两个对象是否相同，因此放入TreeSet的对象必须实现compareTo方法：

class Err{}
public class TreeSetErrorTest{
    public static void main(String[] args){
        TreeSet ts = new TreeSet();
        ts.add(new Err());
        ts.add(new Err());//该程序会报错，因为Err类没有实现compareTo方法，无法自然排序
    }

  • 如果希望TreeSet能正常运作，TreeSet只能添加同一种类型的对象；

  • TreeSet集合判断两个对象是否相等的唯一标准是：两个对象通过compareTo(Object obj)方法比较是否返回0，例如如果一个类重写compareTo方法始终返回一个非零的数，则TreeSet可以添加相同的实例；

  • 重写对象对应类的equals()方法时，应保证该方法与compareTo(Object obj)方法有一致的结果，即两个对象通过equals()方法返回True时，这两个对象通过compareTo(Object obj)方法比较应返回0。

EnumSet的相关性质

  • EnumSet是一个专为枚举类设计的集合类，EnumSet的集合元素有序，EnumSet以枚举值在Enum类内的定义顺序来决定集合元素的顺序，在内部以位向量的形式存储，具有占用内存小、运行效率好的优点，该集合不允许加入null元素；
  • 当试图复制一个Collection集合里的元素来创建EnumSet集合时，必须保证Collection集合里的所有元素都是同一个枚举类的枚举值。
enum Season{
    SPRING, SUMMER, FALL, WINTER;
}

public class EnumSetTest{
    public static void main(String[] args){
        Collection c = new HashSet();
        c.add(Season.SPRING);
        c.add(Season.WINTER);
        EnumSet enumSet = EnumSet.copyOf(c);
        c.add("Hello World");
        enumSet = EnumSet.copyOf(c);//该程序会报错，因为此时HashSet中包含非枚举类对象。
    }

补充说明

HashSet、EnumSet、TreeSet都是线程不安全的，如果有多个线程同时访问一个Set集合，并且有超过一个线程修改了该Set集合，则必须手动保证该Set集合的同步性。通常可以通过Collections工具类的synchronizedSortedSet方法来“包装”该Set集合。（这种同步性的方法性能较低，最好使用concurrentHashMap来提高性能，原因在集合原理篇进行分析）

SortedSet s = Collection.synchronizedSortedSet(new TreeSet(...));

List相关子接口介绍

  • List判断两个对象相等只要通过equals()方法返回true即可，如重写一个类的equals()方法永远返回true，一个列表删除该类的实例（无论是否存在于列表之中）则总是首先删除该列表的第一个元素，字符串的equals方法已经被重写，因此只要两个字符串的排列顺序相同即为同一对象；
  • LinkedList和ArrayList、ArrayDeque底层实现机制不一样，ArrayList和ArrayDeque内部是以数组的形式来保存集合中的元素，因此随机访问集合元素时有较好的性能；而LinkedList内部以链表的形式来保存集合中的元素，因此随机访问性能较差，但在插入、删除元素的时候性能比较出色；
  • LinkedList应该采用迭代器方法来访问更快，而以数组为底层结构的ArrayList和ArrayDeque则以随机访问的方法来访问元素更高效。
